Developer Guide

Integrate TokenSentinel into your applications with the Spring Boot SDK and REST APIs.

Spring Boot SDK

The spring-boot-sdk module provides a Spring Boot Starter that auto-configures REST clients and registers Spring AI @Tool-annotated methods as ToolCallback beans.

Prerequisite Spring Boot 3.4+ with Spring AI 1.4+ on the classpath.

Installation

Build and install the SDK from source:

cd spring-boot-sdk
mvn clean install

Then add it to your project:

<dependency>
  <groupId>com.tokensentinel</groupId>
  <artifactId>token-sentinel-sdk</artifactId>
  <version>1.0.0</version>
</dependency>

Gradle

implementation 'com.tokensentinel:token-sentinel-sdk:1.0.0'

Configuration

Add to application.yml:

tokensentinel:
  api:
    base-url: http://localhost:3001
    api-key: ${AUTH_API_KEY}
  client:
    connect-timeout: 5s
    read-timeout: 10s
    retry:
      max-attempts: 3
      backoff-delay: 1s

Available Tools

The SDK registers 10 tools that Spring AI auto-discovers and exposes to your AI agents:

Tool MethodDescription
getCostSummary(period)Aggregated cost breakdown by model for a time period (1H, 6H, 24H, 3D, 7D)
getModelCosts(period)Per-model cost breakdown for a time period
getAnomalies(limit)Recent anomaly detection events
listTeams()List all configured teams with their budget limits
createTeam(name, monthlyLimit)Create a new team with a monthly token budget
deleteTeam(name)Delete a team and its budget rules
getBudgetStatus(teamName)Get current budget usage for a team
listBudgetRules()List all budget threshold webhook rules
createBudgetRule(...)Create a budget alert rule with team, threshold, and webhook URL
deleteBudgetRule(id)Delete a budget rule by ID

Auto-Configuration

The SDK uses Spring Boot auto-configuration via spring.factories. The following beans are created:

  • TokenCostClient — REST client for cost dashboard APIs (with retry)
  • AdminClient — REST client for admin APIs (teams, budgets)
  • TokenSentinelAutoConfiguration — wires clients and tool provider
  • TokenSentinelAiTools — the 10 @Tool-annotated methods
  • MethodToolCallbackProvider — auto-registers tools with Spring AI

Metrics

The SDK exposes Micrometer metrics:

MetricTypeDescription
tokensentinel.api.requestsCounterNumber of API requests made
tokensentinel.api.errorsCounterNumber of API errors
tokensentinel.api.latencyTimerAPI request latency
tokensentinel.tokens.totalDistributionSummaryToken counts from responses

Usage Example

@Service
public class CostAnalysisService {

    private final TokenSentinelAiTools tools;

    public CostAnalysisService(TokenSentinelAiTools tools) {
        this.tools = tools;
    }

    public String analyzeCosts() {
        String summary = tools.getCostSummary("24H");
        String anomalies = tools.getAnomalies(5);
        return "Cost Summary:\n" + summary + "\n\nAnomalies:\n" + anomalies;
    }
}

Direct REST API Integration

If you're not using Spring Boot, you can call the cost dashboard REST APIs directly:

curl -H "Authorization: Bearer $AUTH_API_KEY" \
  http://localhost:3001/api/dashboard/cost-summary?period=24H

See the API Reference for the full list of endpoints.

Evolutionary Prompt Optimization (EvoluNet-SLM)

TokenSentinel includes an experimental prompt optimizer that uses evolutionary algorithms to discover high-performing prompts.

Running the Optimizer

cd evolunet_slm

# Install dependencies
pip install -r requirements.txt

# Download the base model
python download_model.py

# Run evolution
python evolve.py

Configuration

Key parameters in EvolveConfig:

ParameterDefaultDescription
population_size20Number of prompts per generation
generations10Number of evolutionary generations
mutation_rate0.15Probability of token substitution
crossover_rate0.7Probability of crossover between parents
elite_ratio0.1Top fraction preserved unchanged

Mutation Operators

OperatorDescription
crossover_single_pointSplit both parents at a random token and swap tails
crossover_uniformEach token randomly selected from either parent
mutate_substituteReplace tokens with synonyms from a built-in dictionary
mutate_insertInsert instructional phrases (e.g., "Explain like I'm 5")
mutate_deleteRemove tokens with a given probability
mutate_shuffleSwap adjacent token pairs

Running Tests

cd evolunet_slm
python -m pytest test_evolunet.py -v

31 tests cover all mutation operators, fitness scoring, and the evolution orchestrator.
