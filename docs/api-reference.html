API Reference

Complete reference for all TokenSentinel REST endpoints.

Authentication

All API endpoints (except the dashboard HTML page) require authentication via the Authorization header:

Authorization: Bearer <your-api-key>

The default development key is dev-key-123. Set via AUTH_API_KEY environment variable.

Dashboard Endpoints

Base URL: http://localhost:3001

GET/

HTML cost dashboard page. Does not require authentication.

GET/api/dashboard/cost-summary?period=24H

Aggregated cost breakdown by model.

ParameterTypeDescriptionValid Values
periodstringTime window for aggregation1H, 6H, 24H, 3D, 7D
{
  "total_cost": 1.234,
  "total_requests": 150,
  "total_tokens": 50000,
  "avg_tokens_per_request": 333.3,
  "models": [
    {"model": "gpt-4o-mini", "cost": 0.5, "requests": 100, "tokens": 30000, "pct": 40.5}
  ]
}
GET/api/dashboard/model-costs?period=24H

Per-model cost breakdown.

[
  {"model": "gpt-4o-mini", "cost": 0.5, "requests": 100, "input_tokens": 15000, "output_tokens": 15000}
]
GET/api/dashboard/events

Server-Sent Events stream for real-time cost and anomaly updates. The connection stays open and pushes events as they arrive.

data: {"type": "cost_event", "model": "gpt-4o-mini", "cost_cents": 0.042, "timestamp": "..."}

data: {"type": "anomaly", "model": "gpt-4o", "score": 4.2, "threshold": 3.0}

Prescriptive Engine Endpoints

Base URL: http://localhost:3001

All prescriptive endpoints require authentication via X-Api-Key or Authorization: Bearer <key>.

Assessments

GET/api/prescriptive/assessments

List all prescriptive assessments.

POST/api/prescriptive/assessments

Create a new assessment. Accepts JSON body with cloud infrastructure details and team composition.

{
  "company_name": "My Company",
  "cloud_vendor": "aws",
  "gpu_configs": [{"type": "A100", "count": 8, "region": "us-east-1", "hourly_price": 3.50, "reserved": true}],
  "monthly_request_volume": 1000000,
  "token_distribution": {"input_pct": 0.7, "output_pct": 0.3},
  "current_monthly_spend": 12000,
  "providers_used": [
    {"name": "openai", "models": ["gpt-4o", "gpt-4o-mini"], "monthly_spend": 7000}
  ],
  "team_composition": {"developers": 20, "platform_engineers": 2, "devops": 1, "management": 2},
  "source": "manual"
}
GET/api/prescriptive/assessments/{id}

Get a specific assessment by ID.

PUT/api/prescriptive/assessments/{id}

Update an assessment. Creates a new version.

DELETE/api/prescriptive/assessments/{id}

Delete an assessment and all associated data.

GET/api/prescriptive/assessments/{id}/versions

List all versions of an assessment.

GET/api/prescriptive/assessments/{id}/versions/{version}

Get a specific version snapshot.

POST/api/prescriptive/assessments/{id}/run

Run the prescriptive engine on an assessment. Generates cost breakdown, recommendations, and ROI projections. Stores results in the database.

Reports

GET/api/prescriptive/report/{id}

Returns the assessment report. Serves an interactive HTML page when accessed from a browser, or JSON when accessed via API (based on Accept header).

GET/api/prescriptive/report/{id}/csv

Download the report as CSV with cost breakdown and recommendations.

GET/api/prescriptive/report/{id}/pdf

Download the report as a PDF document with executive summary, cost breakdown table, and top recommendations.

What-If Simulator

POST/api/prescriptive/what-if/{id}

Run a what-if scenario on an existing assessment. Accepts adjustment parameters in the request body:

{
  "volume_multiplier": 1.5,
  "input_pct": 0.6
}

Returns recalculated cost projections based on the adjustments.

CSV Import

POST/api/prescriptive/import/csv

Import cost data from a provider CSV. Supports two modes:

JSON mode: Send a JSON body with column mapping configuration and CSV data as raw text.

Multipart mode: Send a multipart form with the CSV file and column mapping as form fields.

# Multipart form fields:
#   file              — The CSV file
#   assessment_id     — Target assessment ID (optional)
#   has_header        — "true" or "false"
#   model_column      — Column index for model name (0-based)
#   input_tokens_column  — Column index for input tokens
#   output_tokens_column — Column index for output tokens
#   cost_column       — Column index for cost (optional)

Starter Templates

GET/api/prescriptive/templates

Returns three pre-built starter templates: "Startup", "Mid-size", and "Enterprise". Each template includes pre-filled assessment data that can be used to seed a new assessment.

Report Page

Navigate to /api/prescriptive/report/{id} in a browser to view the interactive report page with:

  • Executive Summary with current/projected spend and savings rate
  • Cost Breakdown table by model with team drill-down toggle
  • Recommendations sorted by priority (high/medium/low)
  • What-If Simulator with sliders for request volume and input token ratio
  • Version History with version comparison
  • CSV and PDF download buttons

Continuous Optimization Endpoints

Base URL: http://localhost:3001

All monitoring endpoints require authentication via X-Api-Key or Authorization: Bearer <key>.

Monitoring Rules

Configure spend threshold alerts per model. Runs every 5 minutes.

GET/api/monitoring/rules

List all monitoring rules.

POST/api/monitoring/rules

Create a monitoring rule.

{
  "model": "gpt-4o",
  "pct_threshold": 20,
  "abs_threshold": 100,
  "period": "7d",
  "enabled": true,
  "webhook_url": "https://hooks.slack.com/...",
  "email_to": "admin@company.com"
}
GET/api/monitoring/rules/{id}

Get a monitoring rule.

PUT/api/monitoring/rules/{id}

Update a monitoring rule.

DELETE/api/monitoring/rules/{id}

Delete a monitoring rule.

Alerts

GET/api/monitoring/alerts

List alerts. Filter with query params: ?unacknowledged=true, ?model=gpt-4o, ?type=spend_spike.

POST/api/monitoring/alerts/{id}/acknowledge

Mark an alert as acknowledged (removes from active view).

POST/api/monitoring/alerts/{id}/dismiss

Dismiss an alert permanently.

Savings Tracking

GET/api/monitoring/savings

List auto-detected savings events. The engine detects cost drops >30% vs prior period and records them as savings.

Cost Trends

GET/api/monitoring/trends/{model}?period=7d

Daily cost time-series for a specific model. Used for building cost trend charts.
