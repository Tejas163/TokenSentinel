User Manual

Complete guide to using TokenSentinel — from daily operations to advanced configuration.

Cost Dashboard

The cost dashboard is the primary interface for monitoring AI spend. It is accessible at http://localhost:3001.

Summary Cards

The dashboard header displays four summary metrics for the selected period:

  • Total Cost — aggregate spend across all models
  • Total Requests — number of proxied requests
  • Total Tokens — sum of input + output tokens
  • Avg Tokens/Req — average token consumption per request

Time Period Selection

Toggle between predefined periods: 1H, 6H, 24H, 3D, 7D. The dashboard reloads data on selection change.

Cost by Model

A detailed table shows cost broken down per model, with columns for requests, total tokens, and percentage of total spend.

Live Events (SSE)

When the page is open, a green badge indicates an active SSE connection. New cost events and anomaly alerts stream in real time without page refreshes. A red badge indicates a disconnected state with automatic reconnection.

Anomaly Detection

TokenSentinel detects cost anomalies using a 3-sigma sliding window algorithm:

  • Computed per-model over recent time windows
  • Checked every 5 minutes
  • Anomalies are broadcast via SSE to the dashboard
  • Available via REST: GET /api/admin/anomalies?limit=10
Pro Tip Keep the dashboard open on a monitor for real-time AI cost observability during development.

Per-Team Budgets

TokenSentinel supports per-team budget management for cost allocation across departments or projects.

Creating a Team

curl -X POST http://localhost:3001/api/admin/teams \
  -H "Authorization: Bearer $AUTH_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "engineering", "monthly_token_limit": 10000000}'

Budget-Aware Routing

When a team exceeds its budget, the gateway automatically reroutes to the cheapest available provider. This is triggered via the X-Team-Name request header:

curl -X POST http://localhost:3000/v1/chat/completions \
  -H "Authorization: Bearer dev-key-123" \
  -H "X-Team-Name: engineering" \
  -H "Content-Type: application/json" \
  -d '{"model": "gpt-4o", "messages": [...]}'

Budget Alert Webhooks

Configure webhooks to notify Slack or Teams when a team approaches its budget limit:

curl -X POST http://localhost:3001/api/admin/budget-rules \
  -H "Authorization: Bearer $AUTH_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "team_name": "engineering",
    "threshold_pct": 80,
    "webhook_url": "https://hooks.slack.com/services/..."
  }'

Webhooks fire when usage crosses the threshold percentage. A 30-minute cooldown prevents duplicate notifications.

Slack/Team Cost Digests

Configure scheduled cost digests sent to Slack or Teams webhooks:

VariableDescription
DIGEST_WEBHOOK_URLSlack or Teams incoming webhook URL
DIGEST_SCHEDULECron expression (e.g., 0 9 * * 1 for Monday 9 AM)

Digests include a per-model cost breakdown formatted as Slack message attachments.

Authentication

TokenSentinel uses a shared API key for service-to-service authentication:

  • Client → Rust proxy: HTTP Authorization: Bearer <key> header
  • Rust proxy → Go router: X-Authenticated-User header set by Rust after key validation
  • Dashboard/admin API: Same Authorization: Bearer <key> pattern
  • Dashboard HTML page: Exempt from auth — accessible in any browser

Set the shared key via AUTH_API_KEY environment variable (default: dev-key-123).

Routing Configuration

Routes are stored in Redis as JSON under routes:{pattern}. Each route defines:

{
  "pattern": "/v1/chat/completions",
  "providers": [
    {
      "name": "openai",
      "base_url": "https://api.openai.com",
      "api_key": "${OPENAI_API_KEY}",
      "weight": 3
    },
    {
      "name": "anthropic",
      "base_url": "https://api.anthropic.com",
      "weight": 1
    }
  ]
}

The weight field controls traffic distribution. A provider with weight 3 gets 3x the traffic of a provider with weight 1.

Data Retention

Cost history entries are automatically pruned after 90 days (configurable). The pruning runs periodically on the cost dashboard service.
