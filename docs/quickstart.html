Quick Start

Get TokenSentinel running locally in under 5 minutes.

Prerequisites

  • Docker Compose (v2.20+)
  • Git
  • An API key for an upstream AI provider (e.g., OpenAI, Anthropic)

1. Clone the Repository

git clone https://github.com/codejunkie99/TokenSentinel.git
cd TokenSentinel

2. Configure Environment

Copy the example environment file and edit it:

cp .env.example .env

At minimum, set your upstream provider key:

# .env
OPENAI_API_KEY=sk-your-key-here
AUTH_API_KEY=dev-key-123
Security Note Never commit .env to version control. It is listed in .gitignore.

3. Start All Services

docker compose -f proxyops_gateway/docker-compose.yml up -d

This starts 6 services: rust-proxy, go-router, cost-dashboard, erlang-monitor, redis, and postgres.

4. Verify Everything Is Running

# Check container status
docker compose -f proxyops_gateway/docker-compose.yml ps

# Or run the E2E validation script
# Linux/macOS:
./deploy/run-e2e.sh

# Windows PowerShell:
.\deploy\run-e2e.ps1

The E2E script verifies: health endpoints, cost recordings, the dashboard HTML, SSE events, and data persistence.

5. Send Your First Request

Route through the gateway to your upstream provider:

curl -X POST http://localhost:3000/v1/chat/completions \
  -H "Authorization: Bearer dev-key-123" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o-mini",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'

6. Open the Dashboard

Navigate to http://localhost:3001 in your browser to see the real-time cost dashboard.

7. Run the Demo

A full demo script is available at demo/README.md that walks through services verification, data injection, and cost queries.

cd demo
# Follow the manual steps in README.md
